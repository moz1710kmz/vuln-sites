# VulnSites

**VulnBank / VulnEC / VulnBoard / VulnGraph ＋ コントロールパネル**

VulnSitesは、脆弱性診断の検出トレーニングおよびセキュリティスキャナーの性能評価用に意図的に脆弱化したDocker上で動作するソフトウェアです。
銀行風サイト・EC風サイト・タスク管理SaaS風サイト(SinglePageApplication)・GraphQL の 4 サイトと、それらを束ねるコントロールパネルで構成しています。
4 サイトに **47 件の脆弱性**を散りばめ、コントロールパネルから起動/停止・セキュリティ機能の ON/OFF が可能です。

> ⚠️ **悪用されることを防ぐため、本ソフトウェアを本番環境やインターネットに公開しないでください。ローカル環境、隔離環境専用です。**

## クイックスタート

```bash
docker compose up -d --build
```

| URL | アプリ / 説明 | 脆弱性 |
|-----|------------|--------|
| http://127.0.0.1:8080 | コントロールパネル（`admin` / `panel123`）— ここから各サイトを起動 | — |
| http://127.0.0.1:8081 | **VulnBank** — 銀行業務（送金・口座・ログイン） | 14 |
| http://127.0.0.1:8082 | **VulnEC** — EC（商品・注文・管理画面・3 階層ロール） | 17 |
| http://127.0.0.1:8083 | **VulnBoard** — JWT セッションのタスク管理 SPA | 10 |
| http://127.0.0.1:8084 | **VulnGraph** — GraphQL API（ariadne / 本物のプロトコル） | 7 |

URL のポートは既定値です。アプリ別の合計は 48 件ですが、重複(CSRF)があるのでユニークでは 47 件です。

HTTPS 化・外部公開・ポート変更は `.env` で設定を変更できます（既定値は `.env.example` 参照）。

```env
VULN_TLS=1            # 1 で HTTPS（自己署名）。既定 0=HTTP
BIND_ADDR=0.0.0.0     # 待受アドレス（既定 127.0.0.1）。0.0.0.0 で外部公開。アクセス制限は Security Group / ファイアウォールなど ホスト側で行ってください。
PANEL_PORT=8080       # ホスト側の公開ポート（BANK_APP_PORT などサイト別も同様）
```

停止: `docker compose down`

## コントロールパネルの使い方

1. http://127.0.0.1:8080 にログイン（`admin` / `panel123`）
2. 起動対象（VulnBank / VulnEC / VulnBoard / VulnGraph）とセキュリティ機能を選択して **起動**

---

## セキュリティ機能（バリア）

パネルから個別に ON/OFF できます。脆弱性は消えず、検出難易度だけが上がります。

| 機能 | 内容 |
|------|------|
| `waf` | 素朴な攻撃ペイロードを 403。回避ペイロードあり |
| `otp` | 送金を 2 段階化（OTP 確認） |
| `email_verify` | 登録にメール認証 |
| `csrf` | 全フォームに CSRF トークン必須 |
| `ratelimit` | ログイン失敗でアカウントロック（429） |
| `totp` | ログインに 2 要素認証コード |
| `honeypot` | 隠しフィールドで Bot 検知 |
